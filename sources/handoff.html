10 · Notas para Producción

Backend / Integración

El prototipo usa una base de datos en memoria (const DB = { ... }) que simula todas las operaciones CRUD. En producción, cada función deberá reemplazar sus operaciones sobre DB por llamadas a la API REST correspondiente. Los helpers toast(), openModal() y money() son reutilizables sin cambios.

Persistencia de sesión

La Fase 4C implementa persistencia de sesión en localStorage con clave sg_session y TTL de 7 días. En producción esto debe reemplazarse por tokens JWT con refresh token en httpOnly cookies.

Exportación PDF / Excel

Las funciones exportHistorial('pdf') y window.print() son stubs. En producción: PDF con librería como jsPDF o generación server-side; Excel con SheetJS. El recibo digital ya es imprimible vía window.print().

Iconografía

Todos los íconos son SVG inline referenciados desde el sprite <svg aria-hidden><defs>...</defs></svg> al inicio del body. El set proviene de Lucide Icons (stroke, 24×24). Para producción, usar el paquete npm lucide o el CDN oficial.

Restricciones de la ERS cumplidas

Restricción 2.4 — Alertas no intrusivas: Todos los alert() reemplazados por toasts no bloqueantes con aria-live="polite". Preferencia explícita del encargado en entrevista (NOTAS_SESION.md, pag. 3).
Restricción 2.4 — Multi-dispositivo: Breakpoints implementados para PC, notebook y celulares. Sidebar hamburger + drawer en ≤768px. Layouts apilados en ≤600px.
Pendiente para defensa: Agregar role="dialog" + focus trap en el modal genérico, scope="col" en tablas, y skip-link antes de publicar en producción.
